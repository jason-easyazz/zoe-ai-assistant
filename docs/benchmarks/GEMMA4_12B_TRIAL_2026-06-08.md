# Gemma 4 12B Trial - 2026-06-08

## Outcome

Gemma 4 12B Q4_K_M is not suitable as Zoe's active conversational model on
the current Jetson Orin NX 16GB service footprint.

The known-good Gemma 4 E2B model and llama.cpp build 8662 were restored. The
restored system passes the focused voice suite and retains sub-1.5-second
direct LLM responses.

## Voice Repair

The recurring LiveKit disconnect was caused by two issues in the Jetson
aiortc adapter:

- The adapter did not send LiveKit signaling heartbeats.
- The installed LiveKit protobuf exposes `candidateInit`, while the adapter
  accessed only `candidate_init`, terminating its signal loop on the first ICE
  candidate.

The repair adds both LiveKit ping formats, accepts both protobuf field names,
prevents duplicate agent tasks, and exposes structured pipeline health.

Validation:

- 77 focused voice tests passed.
- One `zoe-agent` remained connected for 30 minutes 44 seconds.
- Zero reconnects, duplicate identities, data-channel aborts, or service
  restarts occurred during the clean soak.
- Five synthetic Kokoro -> Whisper -> E2B -> Kokoro turns passed on the
  upgraded llama.cpp binary.
- Browser HTTP and WebSocket fallback paths completed ten spoken turns during
  certification.

The official native LiveKit Python and Node RTC clients could not initialize a
peer connection in the available headless test environments. The production
aiortc agent connection and browser fallback paths were therefore used for the
acceptance gate.

## Memory And Voice Gate

Full GPU offload failed while allocating a 7024 MiB CUDA model buffer.
llama.cpp auto-fit also failed with attempted contiguous allocations between
5434 and 6614 MiB.

A fixed 24-of-48-layer offload loaded successfully with:

- llama-server RSS: approximately 9.1GB
- available RAM: approximately 2.7GB after the benchmark
- swap: increased from approximately 2.8GB to 3.4GB
- peak observed temperature: below 45C

Five Whisper/Kokoro conversational turns on 12B took 12.5 to 29.3 seconds.
Two of five turns exceeded Zoe's LLM timeout and returned the fallback error.
This failed the usable-voice acceptance gate, so the controller restored E2B
and did not start the 24-hour observation.

A later memory-trim pass stopped OpenClaw, Hermes, and the GitHub runner, then
loaded 36-of-48 layers. Full GPU offload still failed, even with Zoe Data and
Kokoro stopped and approximately 12.7GB reported free by CUDA, while allocating
the same 7024 MiB CUDA buffer. This points to a Jetson/NvMap contiguous
allocation limit rather than ordinary process RSS.

The 36-layer run was faster than the 24-layer run but still failed practical
voice reliability:

- llama-server RSS: approximately 9.1GB
- Kokoro RSS: approximately 2.0GB
- available RAM: approximately 1.5GB to 1.8GB
- swap: approximately 3.1GB before the n-gram/cache test
- short voice pass: one STT failure, one LLM fallback, one successful turn

An n-gram speculative/cache profile was also tested with
`--spec-type ngram-simple`, `--cache-reuse 256`, `--cache-ram 512`,
`--ctx-checkpoints 8`, and 36 GPU layers. It did not improve the text
benchmark and increased swap pressure to approximately 4.8GB, so it was not
kept active.

On 2026-06-11, the official Google Gemma 4 12B IT QAT Q4_0 GGUF was tested:

  `faff1a63667fac17ac5e777f47114688fcefea96e220e211aaa8d62c2c4561f1`
- projector:
  `/home/zoe/models/gemma4-12b-qat/mmproj-gemma-4-12b-it-qat-q4_0.gguf`
- projector SHA-256:
  `e70b0e5cd80323d5d588b4ed06780356b7b1ba03995a4b8164c6ae9db0ff5989`

Full offload, 44 GPU layers, and 40 GPU layers each failed to reach health and
were automatically rolled back. A 36-layer QAT profile loaded successfully with
vision and audio modalities reported by llama.cpp.

The 36-layer QAT profile used:

- llama-server RSS: approximately 8.7GB
- available RAM: approximately 2.0GB to 2.1GB
- swap: approximately 3.8GB to 4.0GB during benchmark
- core side agents stopped except Kokoro, Zoe Data, LiveKit, and OpenClaw

The short Zoe voice pass failed the practical gate:

- "Say hello in one short sentence.": STT failed after 7.7s
- "What is two plus three?": STT failed after 4.3s
- "Give me one reason to drink water.": transcription succeeded after 23.1s,
  but Zoe returned the LLM fallback response

QAT improves some short text timings versus the previous 36-layer Q4_K_M run,
especially arithmetic, but it does not make Gemma 4 12B usable for Zoe's live
voice path on this Jetson configuration.
