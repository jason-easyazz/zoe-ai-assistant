# Brain prefill audit — system prompt + tools

> **IMPLEMENTED** on branch `feat/lean-soul-brevity`. The lean soul, ZOE_SELF
> summary, 8 trimmed tool schemas, and the voice tool/brevity changes all landed
> in `services/zoe-data/zoe_agent.py`. Verified against the live voice replay
> corpus (`~/.zoe-voice-samples`): said-vs-did unchanged (no OK→regression; two
> samples improved EMPTY→OK). The rocks (Gemma 4 E4B+MTP brain, Moonshine v2 STT)
> are untouched — this only trims the *text* reprocessed each turn.
>
> Measured deltas (cl100k_base): `_ZOE_SOUL_STATIC` 3,754→1,532 tok (−59%);
> `_ZOE_SOUL_VOICE` 377→258; the 8 tool schemas 1,645→1,232 (−413); voice tool
> array ~2,350→~1,568 (20→16 tools). The hardcoded panel LAN IP is now config
> (`ZOE_HOST_LAN_IP` env, default 192.168.1.218) injected at import.

## TL;DR

Zoe reprocesses a large fixed prefix on every reply. Measured with `cl100k_base`
against the **live** assembled strings (real `_TOOLS`, real `ZOE_SELF.md` on the host):

| Fixed block | Current | Proposed | Saved |
|---|---:|---:|---:|
| System prompt — `_ZOE_SOUL_BASE` | 2,075 | 1,091 | **−984** |
| System prompt — ZOE_SELF summary | 1,497 | 258 | **−1,239** |
| System prompt — agent-team block | 184 | 184 | 0 |
| **System prompt total (`_ZOE_SOUL_STATIC`)** | **3,754** | **1,533** | **−2,221 (−59%)** |
| Tools — 8 trimmed schemas (subtotal) | 1,645 | 1,077 | **−568** |
| Tools — full 27-tool array | 3,479 | 2,911 | **−568 (−16%)** |

**Per-turn impact (system prompt + tools actually sent):**

| Path | Current | Proposed | Saved |
|---|---:|---:|---:|
| Chat, 1 skill (e.g. weather) | ~4,708 | ~2,176 | **~−2,532 (−54%)** |
| Chat, discovery fallback | ~5,072 | ~2,454 | **~−2,618 (−52%)** |
| Voice, tool turn | ~2,727 | ~1,800 | **~−900 (−34%)** |
| Voice, chit-chat | ~563 | ~480 | ~−83 |

**Headline cut:** kill the duplication. The single biggest waste is that the
system prompt and several tool schemas say the *same thing twice* — the TIER-2
local-research playbook is spelled out in full in both `_ZOE_SOUL_BASE` **and**
the `deep_web_research` description; the ZOE_SELF block is a 1,497-token flat dump
of ~75 MCP tool names, ~34 skill names, and ~23 page names the model never needs
verbatim (the tools it can actually call are already in `_TOOLS`). Collapse the
duplication and the prose, keep every routing rule and behavioral guarantee, and
the fixed prefix drops ~59% with **zero capability loss**.

Nothing is removed. Every one of the 27 tools has a live dispatch handler in
`_dispatch_tool` (verified), so this is trim-not-remove throughout.

## How the prompt + tools are assembled (verified via Serena/codebase-memory)

`services/zoe-data/zoe_agent.py`:

- **`_ZOE_SOUL_BASE`** (L197) — static literal, ~2,075 tok.
- **`_load_zoe_self_summary()`** (L295) — reads `ZOE_SELF.md` (auto-generated by
  agent_sync), truncates to `max_chars=5500`, wraps in `--- ZOE_SELF --- … ---`.
  ~1,497 tok on the live host.
- **`_build_agent_team_prompt()`** (L131 → `zoe_agent_registry.py`) — renders the
  AGENT TEAM block from `agents_registry.yml`. ~184 tok.
- **`_ZOE_SOUL_STATIC = base + self + team`** (L325) — assembled **once at import**.
  `_zoe_soul()` (L435) appends only a ~20-tok dynamic `[datetime] + user` header
  per turn, so the static prefix is KV-cache-reusable. Good design — the win is
  shrinking the reused prefix itself.
- **`_ZOE_SOUL_VOICE`** (L333) — separate trimmed soul for voice (~377 tok).
- **`_TOOLS`** (L456) — 27 OpenAI-format tool dicts; `_HERMES_TOOL` appended (L942).
- **Gating:** `_select_skills()` (L1056) keyword/intent classifier → `_build_tools()`
  (L1092) sends `_ALWAYS_ON_TOOLS` + selected `_SKILL_TOOLS` groups (+ Hermes when
  healthy). `_build_voice_tools()` (L1108) sends `_VOICE_TOOLS` (20 tools) or just
  `_VOICE_ALWAYS_TOOLS`. `FORCE_FULL_CONTEXT=true` is the only path that sends all 27.

## Part 1 — System prompt

See [`zoe-soul-lean-draft.md`](./zoe-soul-lean-draft.md) for the full rewritten
prompt. What's preserved vs cut:

| Element | Current | Draft | Verdict |
|---|---|---|---|
| Identity / "warm, curious, present, not a task executor" | ✓ verbose | ✓ tightened | **kept** |
| Voice rules (contractions, no "Great!/Of course!", gentle takes) | ✓ | ✓ | **kept verbatim intent** |
| Emotional-first / notice-when-off / listening-is-help | ✓ (3 paragraphs) | ✓ (1 paragraph) | **kept, condensed** |
| "Answer everyday questions from own knowledge" | ✓ | ✓ | **kept** |
| Never-simulate bash/python | ✓ | ✓ | **kept** |
| Function-call mechanism / no JSON-in-text | ✓ (twice) | ✓ (once) | **kept, de-duped** |
| TOOL ROUTING list (weather/calendar/reminder/list/memory/ha/proactive/bash) | ✓ | ✓ | **kept all** |
| VISUAL TOOLS (show_map/show_chart/show_action_menu/setup_telegram/list_openclaw_*) | ✓ separate section | ✓ folded into routing | **kept all rules** |
| SELF-BUILDING (builder-skill highlights + Hermes) | ✓ | ✓ | **kept** |
| WEB SEARCH 3-tier playbook | ✓ (~40 lines) | ✓ (~6 lines) | **kept all 3 tiers + query rules** |
| ESCALATION rules (do/don't escalate) | ✓ | ✓ | **kept** |
| TOUCH PANELS (panel_* tools, LAN IP, no-Cloudflare) | ✓ | ✓ | **kept** |
| Worked Discord example | ✓ full | ✓ one-liner | **kept (compressed)** |
| ZOE_SELF: tier table + capabilities + escalation guide | ✓ | ✓ | **kept (as summary)** |
| ZOE_SELF: 75-name MCP tool dump | ✓ | ✗ | **cut — redundant with `_TOOLS`** |
| ZOE_SELF: 34-name skill dump, 23-name page dump, ports, A2A, Multica internals | ✓ | ✗ | **cut — never needed verbatim in-prompt** |

**Why nothing functional is lost:** every behavioral instruction, tool-routing
trigger, escalation gate, and the full voice/personality survive. The cuts are
(a) prose verbosity, (b) the same instruction stated twice, and (c) raw
name-list dumps that don't drive behavior — the model can only *call* the tools
in `_TOOLS`, and capability-gap awareness is preserved by the kept summary +
`list_openclaw_skills` / `escalate_to_hermes` fallbacks.

### ZOE_SELF caveat (implementation note)

`ZOE_SELF.md` is **auto-generated by agent_sync**, so don't hand-edit the doc.
The lean-summary win lands in `_load_zoe_self_summary()`: lower `max_chars`
(e.g. 5500 → ~1200) and/or summarize to the tier-table + capabilities + escalation
sections, dropping the flat name dumps. This is a code change for the approval
pass, sketched in the draft. If preferred, ship Part-1-base first (−984 tok, no
generator coupling) and treat the ZOE_SELF trim as Phase 2.

## Part 2 — Tools

**No removals.** All 27 verified live in `_dispatch_tool`. Two levers:

### (a) Tighter per-turn gating — voice path

`_VOICE_TOOLS` sends 20 tools (2,350 tok) whenever `_voice_needs_tools()` is true.
Recommend trimming the voice recovery set to the genuinely voice-reachable tools and
relying on `escalate_to_hermes` for the long tail:

- Drop from voice: `escalate_to_openclaw` (187), `list_openclaw_plugins`-class
  discovery, `mempalace_add` (covered by intent router + `memory_update`), and the
  duplicate of `show_action_menu` if not voice-rendered.
- Combined with leaner schemas (below), the voice tool turn drops from ~2,350 to
  ~1,500 tok — closer to the "compact for latency" intent the code comments claim.

Chat gating (`_build_tools`) is already tight; no change needed beyond schema trims.

### (b) Leaner schemas (measured)

| Tool | Current | Lean | Saved | What changed |
|---|---:|---:|---:|---|
| `deep_web_research` | 409 | 150 | **−259** | Drop the 10-bullet examples block (duplicated in prompt); keep intent + "include full location" + 1 example |
| `show_chart` | 222 | 187 | −35 | Trim per-field prose |
| `show_map` | 217 | 145 | −72 | Trim "Use when…" prose, drop redundant field docs |
| `list_openclaw_skills` | 148 | 113 | −35 | Compress the highlight instruction |
| `show_action_menu` | 181 | 135 | −46 | One-line usage rule |
| `proactive_schedule` | 167 | 128 | −39 | Compress |
| `memory_update` | 162 | 127 | −35 | Compress |
| `web_search` | 139 | 92 | −47 | Compress tier-routing note |
| **Subtotal** | **1,645** | **1,077** | **−568** | |

These are description-only trims — parameter *names, types, enums, and required
lists are unchanged*, so tool-calling accuracy is preserved (the model keys on the
schema shape, not the prose). The `deep_web_research` example bullets are fully
covered by the prompt's TIER-2 section, so removing them from the schema removes a
verbatim duplicate, not information.

## Rollout (after approval — separate PR, touches `zoe_agent.py`)

1. **Phase 1 (lowest risk):** swap `_ZOE_SOUL_BASE` for the lean draft (−984 tok).
   No generator coupling. A/B against the live voice replay corpus + a chat sanity set.
2. **Phase 2:** trim `_load_zoe_self_summary()` (−1,239 tok). Validate capability-gap
   answers still surface builder skills / Hermes.
3. **Phase 3:** apply the 8 leaner tool schemas (−568 tok) and the voice-set trim.
   Re-run the live voice replay harness `services/zoe-data/tests/replay_samples.py`
   (replays Jason's `~/.zoe-voice-samples` corpus, said-vs-did diff) — a regression
   there blocks. If a chat-side equivalent doesn't exist yet, the gate is a manual
   routing check: a fixed set of prompts (one per skill group + the discovery
   fallback) must produce identical tool-selection before/after.

Acceptance bar: the voice replay corpus and a chat routing set must show identical
tool-selection and identical said-vs-did behavior before/after each phase.

**Bonus fix to fold in (Greptile P2):** today's `_ZOE_SOUL_BASE` hardcodes the
server LAN IP `192.168.1.218` in the static prefix — stale on any host move,
fixable only by re-editing + restarting. The lean draft uses a `{ZOE_HOST_LAN_IP}`
placeholder; the approval PR should inject it from an env var into the **dynamic**
header (`_zoe_soul()`), so a host change becomes config, not code.

## What is NOT touched

- Models (Gemma 4 E4B+MTP, Moonshine v2 Medium) — rocks, untouched.
- Skill→tool gating *logic*, intent router, KV-cache prefix ordering — unchanged.
- Tool *parameters* (names/types/enums/required) — unchanged; descriptions only.
- `zoe_agent.py` itself in this PR (docs-only).
