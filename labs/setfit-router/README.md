# setfit-router — classifier head on Zoe's bge-small embeddings (LAB SPIKE)

Raises fast-tier routing coverage by training a supervised classifier head on
the **same frozen `BAAI/bge-small-en-v1.5` embedding** that
`services/zoe-data/semantic_router.py` already computes per turn (fastembed
ONNX, CPU). LAB-ONLY: nothing here is wired into `fast_tiers.py`, prod config,
or the voice path — this directory delivers trained heads + numbers + an
integration plan.

## VERDICT: **ADOPT (logreg head, threshold ≈0.4)** — +13–19 pts overall, paraphrases 35.9% → 56–64%, at ~0.2 ms added latency

## Results — held-out 81-case needle-benchmark corpus (labs/needle-benchmark PR #1276; NEVER trained on)

| router | overall | canonical | paraphrase | chat-FP (chat turns misrouted to tools) | added latency |
|---|---|---|---|---|---|
| Baseline: Tier-0 regex + Tier-1 similarity (PR #1276) | 61.7% | 85.3% | **35.9%** | 12.5% | regex 0.2ms / embed ~60ms |
| **logreg head** (raw argmax) | **79.0%** | 94.1% | **64.1%** | 12.5% | **+0.19 ms** (embedding already computed) |
| **mlp head** (raw argmax) | **80.2%** | 97.1% | 64.1% | 12.5% | +14.4 ms |
| logreg @ conf ≥ 0.4 (abstain → brain) | 75.3% | — | 56.4% | **0.0%** | +0.19 ms; decides 78.1% of tool turns |
| mlp @ conf ≥ 0.7 (abstain → brain) | 77.8% | — | 59.0% | **0.0%** | +14.4 ms; decides 87.7% of tool turns |

The trade to call out: **raw argmax maximises accuracy but misroutes 1-in-8
chat turns into a tool** (disqualifying for Samantha-grade conversation — the
same failure that sank Needle). Adding a confidence gate buys **0% chat-FP**
at the cost of a few paraphrase points: logreg@0.4 keeps 56.4% paraphrase
(still +20 pts over baseline) while deciding 78% of tool turns itself;
**mlp@0.7 is the best gated operating point** (77.8% overall / 59.0%
paraphrase / 0% chat-FP / 87.7% tool coverage) if 14 ms is acceptable.
Recommended prod point: **logreg @ 0.4** — within 2 pts of gated mlp at 75×
less head latency and a 38 KB artifact; revisit mlp if coverage matters more.

Remaining errors are dominated by genuinely brain-worthy indirection
("we're out of laundry powder again" → lists; "mum's flight lands sunday 6pm,
stick that in" → calendar) plus notes↔memory boundary confusion — see
`results/eval.json` `errors`.

- 5-fold CV on the training set: logreg 0.904 ± 0.026, mlp 0.906 ± 0.020.
- Artifacts: `artifacts/head_logreg.joblib` (38 KB), `artifacts/head_mlp.joblib` (1.5 MB) — both committed.

## Label set (13 classes)

The 9 live `semantic_router.ROUTES` domains **plus the domains ROUTES is
missing today** — `notes`, `journal`, `music`, `smart_home` — and an explicit
`chat` none-class (conversational turns must NOT route to tools). See
`labels.py`.

## Training data

`data/train.jsonl` — 1,121 examples, 82–94 per class (target band 50–150/class
met for all 13 classes):

- **seeds**: the live `semantic_router.ROUTES` utterances verbatim + hand
  seeds for the 4 missing domains (phrasing mined from `intent_router.py`
  regexes and the real tool descriptions in `labs/needle-benchmark/zoe_tools.json`);
- **synthetic paraphrases**: generated by the live local Gemma brain
  (llama-server :11434, OpenAI-compatible, sequential + 2 s pause — gentle on
  live traffic) across 4 style prompts including sloppy/indirect and
  ASR-style lowercase; chat includes hard tool-vocabulary negatives;
- dedup-normalized, and **filtered against the 81 eval texts** so nothing in
  the held-out corpus leaks into training. (`~/.zoe-voice-samples` was checked
  read-only: it contains WAVs with no transcript sidecars, so no usable text
  seeds there.)

## On "SetFit-style" contrastive fine-tuning

Deliberately **not** done as a body fine-tune: the whole premise of this head
is consuming the embedding `semantic_router` already computes per turn. A
contrastively fine-tuned bge body is a *different* model — prod would have to
run a second 33M-param encoder per turn, forfeiting the ~0-latency win and
doubling embed RAM on a memory-tight box. The frozen-embedding logreg/mlp
heads capture most of the achievable gain (CV ~0.90); revisit body
fine-tuning only if the gated head plateaus in shadow mode.

## Reproduce

```bash
cd labs/setfit-router
python3 build_dataset.py   # regenerates data/train.jsonl via llama-server :11434 (resumable; skip — file is committed)
python3 train.py           # embeds via fastembed bge-small-en-v1.5 (prod model), writes artifacts/
python3 eval.py            # evaluates on eval/needle_corpus.jsonl, writes results/eval.json
```

CPU-only; deps: fastembed 0.8.0, scikit-learn 1.7.2, joblib, numpy (all on-box).
`eval/needle_corpus.jsonl` is a verbatim copy of `labs/needle-benchmark/corpus.jsonl`
(PR #1276) so this lab stands alone.

## Integration plan for fast_tiers.py (PLAN ONLY — no prod change in this PR)

1. **Where it slots in**: inside `semantic_router.route()` (or a thin wrapper
   called from `fast_tiers.resolve()` Tier-1). The utterance embedding `v` is
   already computed there; add
   `proba = head.predict_proba(v.reshape(1,-1))` → `domain, conf`. Load the
   joblib head lazily next to the ROUTES matrix in `_ensure_loaded()`
   (38 KB, no new RAM concern).
2. **Decision rule**: `routed = domain if (conf >= ZOE_ROUTER_HEAD_THRESHOLD
   and domain != "chat") else "chat"` — i.e. below-threshold and chat both
   fall through to the brain exactly like today's `route()` contract, so
   `fast_tiers.resolve()` needs **no signature change**; the existing
   `scores`/margin fields can carry `proba` per class, keeping the
   `_router_margin()` ambiguity check working unchanged.
3. **New domains need experts**: the head emits `notes`, `journal`, `music`,
   `smart_home`, which `expert_dispatch` must handle (music/smart_home wiring
   is the known skybridge gap). Until an expert exists, keep those domains in
   the channel `defer_domains` list → brain, so the head's routing gain lands
   domain-by-domain as experts arrive.
4. **Rollout**: ship behind `ZOE_ROUTER_HEAD=shadow` first (log head decision
   + agreement alongside today's similarity decision, same pattern
   semantic_router used); flip to active per-channel after live shadow
   agreement confirms the offline numbers. Start `ZOE_ROUTER_HEAD_THRESHOLD=0.4`
   (logreg). Voice-path change ⇒ replay-gate against `~/.zoe-voice-samples`
   before deploy (mandatory).
5. **Tier-0 regex stays** in front (100%-precise hits at 0.2 ms); the head
   only replaces the max-cosine-similarity Tier-1 decision.
